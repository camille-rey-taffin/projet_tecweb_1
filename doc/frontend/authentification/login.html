Login¶

ressource : /login

get¶

Affiche la page permettant l’authentification via un formulaire si l’utilisateur n’est pas connecté (si pas de token dans la session). Affiche un message d’accueil sinon. Le bouton « connecter » déclenche une requête POST sur la même ressource.

Login (utilisateur non connecté)¶

Login (utilisateur connecté)¶

Paramètres :

Champ

Valeurs possibles

Description

logout (optionnel)
Boolean :
True / False
Indique que l’utilisateur vient de se déconnecter
Déclenche l’affichage d’un message de confirmation de la déconnexion

error (optionnel)
string :
invalid_token
login_required
Indique l’erreur à l’origine de la redirection vers la page de login
Déclenche l’affichage d’un message d’explication correspondant à l’erreur au-dessus du formulaire

next (optionnel)
string :
URL d’une ressource de l’application
Indique la page vers laquelle rediriger après login (utilisée dans la méthode POST)


Exemple de requête :

ressource : /login?error=login_required&next=%2Fdata%2Fsearch

L’utilisateur vient d’être redirigé vers la page de login après avoir essayé d’accéder à la page /data/search sans être authentifié.

Réponses :

Code

Type

Description

200

html/text

Page html contenant le formulaire de login ou un message d’accueil

401

html/text

Page html contenant le formulaire de login précédé d’un message d’erreur


post¶

Méthode déclenchée par la soumission du formulaire de login. Intéragit avec l’API pour authentifier l’utilisateur. Si l’authentification aboutit, token d’authentification, id d’utilisateur, nom et prénom d’utilisateurs sont ajoutés à la session. Sinon un message d’erreur est affiché.

Paramètres (formulaire):

Champ

Valeurs possibles

Description

name (obligatoire)

string

Le nom de l’utilisateur à authentifier

firstnamename(obligatoire)

string

Le prénom de l’utilisateur à authentifier

next - URL (optionnel)
string :
URL d’une ressource de l’application

Indique la page vers laquelle rediriger après login. (utilisée dans la méthode POST)

Paramètres (URL) :
